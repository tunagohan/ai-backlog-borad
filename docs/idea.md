# Idea (input)

## 何を作りたい？
- 商業施設や他店舗向けオーナーに対して日々の点検業務などをDXするSasS

## 想定ユーザー / 誰の痛み？
- オーナー向け
  - 業務に対しての週次、月次、年次の点検報告書類を確認するのが大変
  - いつどこでどんな不具合が発生していたのかを複数の物件を持っている場合に管理しきれない
- 点検作業者向け
  - 紙媒体での報告となっており、すぐに報告することが難しい


## ほしい機能（雑でOK）
- 会社を登録することが出来る
- 会社に紐づくユーザーを登録することが出来る
- 会社に紐づく物件を登録することが出来る
- 物件に紐づく店舗を登録することが出来る
  - 店舗に紐づくスペースを登録することが出来る
  - スペースに紐づく資産を管理することが出来る
    - 資産というのは電球やエアコン、室外機などあらゆる点検対象となるもの
- 物件、スペースなどを指定し、点検などを対象とした業務を登録することが出来る
  - 業務を実施することが出来る
  - 業務内容は大項目、項目に分かれており、項目の中に良否、数値の選択を行い作成
  - これらの業務は業務設定画面で作成を行い、その設定を業務に適用するイメージ
- 業務中に不具合がある場合、不具合を報告することが出来る

## 制約（技術/期限/コスト/法務など）
- 技術: Rails + Nuxt
- コストは無料の範囲で
- 作り込まなくてもいいので簡単に一連の会社登録から業務報告->不具合報告まで出来ると良い
  - そのためユーザー登録導線などは一旦カットし、ダミーユーザーでの実施などでも可能
- デザインは今どきのデザインでシンプルで丸みのあるデザインをベースにする
  - PC/スマホのレスポンシブ対応
  - カラーは点検を行なうような青と緑をベースにしたい
