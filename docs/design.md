# Design

## User flow
1. 初期表示（ダミーユーザーで利用開始）
2. 会社登録
3. 物件登録
4. 店舗登録
5. スペース登録
6. 資産登録
7. 点検テンプレート作成（大項目/項目、良否 or 数値）
8. 点検業務を対象（物件/スペース）へ割り当て
9. 点検実施（項目入力、必要なら不具合報告）
10. 点検完了
11. オーナーがダッシュボードで直近点検結果と未解決不具合件数を確認
12. 不具合一覧で追跡

補足:
- 作業者導線は「業務選択 -> 入力 -> 完了」を最短化し、現場での入力負荷を下げる。
- オーナー導線は「状況把握 -> 問題特定」を優先し、一覧可読性を重視する。

## Screens
1. 会社/階層セットアップ
 - 目的: 会社、物件、店舗、スペース、資産を登録して点検対象を定義する。
 - 主要要素: 階層ナビ、登録フォーム、登録済み一覧。
2. 点検テンプレート作成
 - 目的: 点検項目を標準化する。
 - 主要要素: 大項目追加、項目追加（良否/数値）、テンプレート保存。
3. 点検業務一覧・開始
 - 目的: 割り当て済み業務から対象業務を選択して開始する。
 - 主要要素: 対象フィルタ（物件/スペース）、ステータス、開始ボタン。
4. 点検実施
 - 目的: 項目入力と不具合報告を行い、業務を完了する。
 - 主要要素: 項目カード、入力UI（良否トグル/数値入力）、不具合報告モーダル、完了ボタン。
5. 不具合一覧
 - 目的: 報告済み不具合を対象単位で追跡する。
 - 主要要素: 不具合カード/テーブル、対象情報、発生日、ステータス。
6. オーナーダッシュボード
 - 目的: 物件別の点検状況と未解決不具合を俯瞰する。
 - 主要要素: 物件別サマリー、直近点検結果、未解決件数。

## States
- empty
- loading
- error

画面別の状態定義:
1. 会社/階層セットアップ
 - empty: 「まずは会社情報を登録してください」
 - loading: 一覧とフォーム送信中のスケルトン表示
 - error: 「登録に失敗しました。入力内容を確認して再度お試しください」
2. 点検テンプレート作成
 - empty: 「テンプレートがありません。最初のテンプレートを作成しましょう」
 - loading: テンプレート保存中インジケーター
 - error: 「テンプレート保存に失敗しました。時間をおいて再試行してください」
3. 点検業務一覧・開始
 - empty: 「実施可能な業務がありません」
 - loading: 業務一覧の取得中表示
 - error: 「業務一覧を取得できませんでした」
4. 点検実施
 - empty: 「点検項目が未設定です。テンプレートを確認してください」
 - loading: 入力保存中（自動保存/完了処理）
 - error: 「入力の保存に失敗しました。通信環境を確認してください」
5. 不具合一覧
 - empty: 「現在、未解決の不具合はありません」
 - loading: 不具合一覧の取得中表示
 - error: 「不具合一覧を取得できませんでした」
6. オーナーダッシュボード
 - empty: 「表示する点検データがありません」
 - loading: サマリー集計中表示
 - error: 「ダッシュボードの読み込みに失敗しました」

## Copy (JP)
主要コピー:
- グローバル
  - 画面タイトル: 「点検業務管理」
  - 主CTA: 「保存する」「点検を開始」「点検を完了」
  - セカンダリCTA: 「下書き保存」「キャンセル」
- 会社/階層セットアップ
  - 見出し: 「点検対象を登録」
  - 説明: 「会社・物件・店舗・スペース・資産を順に登録します」
  - 空状態: 「まだ登録がありません。右上の「新規登録」から開始してください」
- 点検テンプレート作成
  - 見出し: 「点検テンプレートを作成」
  - 説明: 「大項目と項目を設定し、現場で使う点検票を作ります」
  - 成功トースト: 「テンプレートを保存しました」
- 点検実施
  - 見出し: 「点検を実施」
  - 説明: 「各項目を入力し、必要に応じて不具合を報告してください」
  - バリデーション: 「未入力の項目があります」
  - 完了確認: 「この内容で点検を完了しますか？」
  - 完了後: 「点検結果を保存しました」
- 不具合報告
  - モーダルタイトル: 「不具合を報告」
  - 項目ラベル: 「発生箇所」「内容」「緊急度」
  - 送信成功: 「不具合を登録しました」
- ダッシュボード
  - 見出し: 「オーナーダッシュボード」
  - 指標ラベル: 「直近点検」「未解決不具合」
