# PO / PRD

## Goal
- 商業施設オーナーが、複数物件の点検業務と不具合報告を1つの画面群で把握できるようにする。
- 点検作業者が紙を使わず、現場で点検結果と不具合を即時入力できるようにする。
- MVPでは「会社登録 -> 物件/店舗/スペース/資産登録 -> 点検業務実施 -> 不具合報告」までの一連フローを成立させる。

## Non-goals
- 高度な権限管理（細かなロール設計、承認ワークフロー）は対象外。
- 外部システム連携（会計、BI、通知基盤、IoT連携）は対象外。
- 請求/課金、契約管理、監査証跡の厳密運用は対象外。
- 完全な帳票自動生成（法定フォーマット対応）は対象外。

## User stories
- オーナーとして、会社と物件階層を登録し、どの物件で何が点検対象かを一覧で把握したい。
- オーナーとして、点検実施履歴と不具合履歴を物件単位で確認し、見落としを減らしたい。
- 点検作業者として、現場で点検業務を選択し、良否/数値を入力してすぐ報告したい。
- 点検作業者として、点検中に見つけた不具合を対象（物件/店舗/スペース/資産）に紐づけて報告したい。

## Acceptance Criteria (AC)
1. 会社を1件以上登録できる。
2. 会社配下に物件、物件配下に店舗、店舗配下にスペース、スペース配下に資産を登録できる。
3. 点検業務テンプレート（大項目/項目、項目種別: 良否または数値）を1件以上作成できる。
4. 登録済みの対象（物件/スペース等）に業務を割り当てて実施開始できる。
5. 実施画面で各項目に対して必須入力を満たさないと完了できない。
6. 業務完了後、実施日時・実施者（MVPではダミーユーザー可）・入力結果が保存される。
7. 実施中または実施後に不具合を登録でき、対象階層に紐づけて一覧表示できる。
8. オーナー向け画面で、少なくとも「物件別の直近点検結果」と「未解決不具合件数」を確認できる。
9. UIはPC/スマホで表示崩れなく利用できる（主要画面: 登録、実施、一覧）。

## MVP scope
- In scope
  - 会社/物件/店舗/スペース/資産のCRUD（最低Create/Readは必須）
  - 点検業務テンプレート作成と業務実施
  - 不具合報告の登録/一覧
  - オーナー向けの簡易ダッシュボード（直近結果、不具合件数）
  - ダミーユーザー前提での操作（ログイン導線は簡略化）
- Out of scope
  - 本番想定の認証/認可
  - 通知（メール/Push/Slack）
  - 高度な検索、分析、CSV/PDFエクスポート
  - マルチテナント分離の厳密実装

## Metrics / Logging
- Product metrics
  - 1日あたり点検完了件数
  - 不具合報告件数（新規/未解決）
  - 点検完了率（割当業務に対する完了比率）
  - 平均報告リードタイム（業務開始から完了まで）
- Event logging (minimum)
  - `company_created`
  - `asset_registered`
  - `inspection_template_created`
  - `inspection_started`
  - `inspection_completed`
  - `issue_reported`
  - `owner_dashboard_viewed`
